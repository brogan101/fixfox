Short description (150-300 chars):
Fix Fox is a local-first Windows troubleshooting app with guided runbooks, safe fixes, evidence bundles, and validated Home/Ticket support pack exports. Built for home users and MSP/IT workflows without cloud lock-in.

Long description:
Fix Fox helps you diagnose, document, and resolve common Windows issues with a structured, professional workflow.

What it does:
- Home-first troubleshooting goals for Wi-Fi, storage, speed, browser, and printer issues.
- IT-ready runbooks for ticket triage, update repair, network stack repair, and integrity checks.
- ToolRunner execution model with progress, cancel, logs, and deterministic next steps.
- Support pack exports with report, findings CSV, actions logs, evidence bundles, manifest, and hashes.

Why it is different:
- Local-first by default: no forced cloud pipeline.
- Share-safe masking support across copied/exported text surfaces.
- Validation-first exports for reliable handoff quality.
- Minimal UI clutter: focused nav + searchable directories + command palette.

Feature bullets (Home + MSP):
- Home goal cards with guided mini-playbook workflows
- Wi-Fi wizard with DNS/proxy/hosts/WLAN context + safe DNS flush
- Storage radar with bounded scan + ranked bars + downloads preview plan
- PC slow sampler with sustained CPU/RAM sampling + startup inventory
- Printer rescue with spooler/queue/PrintService event export
- Browser rescue with default-browser/proxy/DNS/version context
- IT evidence bundles (system/network/updates/crash/eventlogs)
- Ticket Pack export with validator + manifest + hashes
- History timeline with reopen/re-export/compare
- Run Center with recent run status and rerun/export actions

FAQ:
Q1: Does Fix Fox require internet?
A1: No. Core diagnostics, runbooks, and export flows run locally.

Q2: Does it upload my data?
A2: Not by default. Data is stored locally under %LOCALAPPDATA%\FixFox.

Q3: What is a Home Share Pack?
A3: A lighter export preset for routine sharing with safer defaults.

Q4: What is a Ticket Pack?
A4: A richer support package with core evidence bundles for IT/MSP tickets.

Q5: Can I run without admin rights?
A5: Yes. Safe tasks work without admin; admin tasks are explicitly gated.

Q6: Are files deleted automatically?
A6: No silent deletion. Cleanup workflows are preview-first.

Q7: Can I cancel long operations?
A7: Yes. Long tasks run in workers with cancel/timeouts via ToolRunner.

Q8: How do I find tools quickly?
A8: Use global search or Ctrl+K command palette.

Q9: Is there a rollback strategy?
A9: Fix detail includes rollback guidance, and reversible actions are surfaced in rollback flows.

Q10: What if export validation fails?
A10: Export is blocked by default; a confirmed override path is available.

Q11: Can I re-open old sessions?
A11: Yes. History supports reopen, compare, and re-export without rerun.

Q12: Is this suitable for MSPs?
A12: Yes. Ticket-focused runbooks and evidence bundles are built for support handoff quality.

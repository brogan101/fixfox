# FixFox Support Packages

Support packages exist to reduce guesswork when self-service is not enough.

## What you get

- a readable summary for a technician or support person
- a technical details file for deeper review
- recent repair receipts with verification and next-step notes
- device health and issue context

## Good times to generate one

- a repair says it failed or verification did not improve
- FixFox keeps recommending escalation
- the issue has crossed from local troubleshooting into account, permissions, policy, or certificate territory
- you want proof of what FixFox already tried

## Before you share a package

1. Read the preview inside FixFox.
2. Confirm the issue summary is accurate.
3. Choose the right detail level in Settings if you need deeper technician context.
4. Remove or replace any extra notes you would not want to share externally.

## Where packages are stored

Packages are written into the local FixFox app-data folder and named by timestamp so multiple support sessions do not overwrite each other.

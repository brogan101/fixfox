# FixFox Recovery and Resume

FixFox keeps interrupted work visible so a restart, crash, or forced close does not leave you guessing.

## If FixFox did not close cleanly

On the next launch, FixFox checks whether the previous session ended cleanly.

- fresh, resumable guided work can be restored
- stale interrupted state is kept for review instead of being resumed blindly
- you can dismiss interrupted work from the Support Package page when it is no longer relevant

## If a repair fails

FixFox should tell you:

- what step failed
- what did not change
- whether verification passed
- whether rollback is available
- what the safest next step is

## If reboot is required

Use the receipt and support summary as your handoff point. After reboot:

- reopen FixFox
- review the last interrupted or failed work
- continue only if the workflow is still relevant and safe

## When to stop using FixFox and escalate

- Windows startup or recovery actions become the next step
- the issue points to account or permissions ownership
- the device is locked down by policy
- the failure points to reinstall or possible hardware trouble
